# Genome Format

A flame fractal genome is a JSON document describing the iterated function
system (IFS) that the chaos-game compute shader evaluates. The Rust types
live in `src/genome.rs`.

## FlameGenome

Top-level struct. Serialized as `{name}.json` in the genomes directories.

| Field | Type | Description |
|---|---|---|
| `name` | `String` | Unique identifier (e.g. `"child-4821"`, `"mutant-7193"`) |
| `global` | `GlobalParams` | Rendering parameters: `speed`, `zoom`, `trail`, `flame_brightness` |
| `kifs` | `KifsParams` | Kaleidoscopic IFS params: `fold_angle`, `scale`, `brightness` |
| `transforms` | `Vec<FlameTransform>` | The transform set (typically 3-6) |
| `final_transform` | `Option<FlameTransform>` | Optional post-iteration transform applied after each chaos step |
| `symmetry` | `i32` | Positive = rotational symmetry order, negative = bilateral symmetry (default: 1) |
| `palette` | `Option<Vec<[f32; 3]>>` | 256-entry RGB color table, or `None` for cosine fallback |
| `parent_a` | `Option<String>` | First parent's name (lineage tracking) |
| `parent_b` | `Option<String>` | Second parent's name (lineage tracking) |
| `generation` | `u32` | Breeding depth from original seed (increments each breed) |

### GlobalParams

| Field | Type | Description |
|---|---|---|
| `speed` | `f32` | Animation speed |
| `zoom` | `f32` | Camera zoom level (auto-set by `auto_zoom()`) |
| `trail` | `f32` | Trail persistence (overridden by `RuntimeConfig.trail`) |
| `flame_brightness` | `f32` | Base brightness multiplier |

### KifsParams

| Field | Type | Description |
|---|---|---|
| `fold_angle` | `f32` | Kaleidoscopic fold angle |
| `scale` | `f32` | KIFS scale factor |
| `brightness` | `f32` | KIFS brightness contribution |

## FlameTransform

Each transform defines one function in the IFS. The chaos game randomly
selects transforms weighted by `weight`, applies the affine matrix, then
blends variation functions.

### Core fields

| Field | Type | Description |
|---|---|---|
| `weight` | `f32` | Selection probability (normalized to sum to 1.0 across transforms) |
| `a` | `f32` | Affine coefficient (row 0, col 0) -- default 1.0 |
| `b` | `f32` | Affine coefficient (row 0, col 1) -- default 0.0 |
| `c` | `f32` | Affine coefficient (row 1, col 0) -- default 0.0 |
| `d` | `f32` | Affine coefficient (row 1, col 1) -- default 1.0 |
| `offset` | `[f32; 2]` | Translation (x, y) |
| `color` | `f32` | Palette index (0.0-1.0), indexes into the 256-entry palette |

The affine transform computes:
```
x' = a*x + b*y + offset[0]
y' = c*x + d*y + offset[1]
```

### Variation weight fields

All 26 variation functions, each a `f32` weight (0.0 = inactive). After
normalization, the active variation weights sum to 1.0 with at most 2
dominant variations per transform.

| Index | Field | Index | Field |
|---|---|---|---|
| 0 | `linear` | 13 | `spiral` |
| 1 | `sinusoidal` | 14 | `diamond` |
| 2 | `spherical` | 15 | `bent` |
| 3 | `swirl` | 16 | `waves` |
| 4 | `horseshoe` | 17 | `popcorn` |
| 5 | `handkerchief` | 18 | `fan` |
| 6 | `julia` | 19 | `eyefish` |
| 7 | `polar` | 20 | `cross` |
| 8 | `disc` | 21 | `tangent` |
| 9 | `rings` | 22 | `cosine` |
| 10 | `bubble` | 23 | `blob` |
| 11 | `fisheye` | 24 | `noise` |
| 12 | `exponential` | 25 | `curl` |

### Parametric variation params

Stored in `variation_params: HashMap<String, f32>`. These control the
behavior of specific parametric variations:

| Key | Default | Used by |
|---|---|---|
| `rings2_val` | 0.5 | `rings` variation |
| `blob_low` | 0.2 | `blob` variation (minimum radius) |
| `blob_high` | 1.0 | `blob` variation (maximum radius) |
| `blob_waves` | 5.0 | `blob` variation (wave count) |
| `julian_power` | 2.0 | `julia` variation (power parameter) |
| `julian_dist` | 1.0 | `julia` variation (distance parameter) |
| `ngon_sides` | 4.0 | N-gon variation (number of sides) |
| `ngon_corners` | 2.0 | N-gon variation (corner sharpness) |

### GPU buffer layout

Each transform is packed into 42 floats for the storage buffer:

```
[0]  weight
[1]  a  [2] b  [3] c  [4] d
[5]  offset_x  [6] offset_y
[7]  color
[8-33]  26 variation weights (linear through curl)
[34-41] 8 parametric variation params (rings2_val, blob_low, blob_high,
        blob_waves, julian_power, julian_dist, ngon_sides, ngon_corners)
```

## Palette

- 256 entries of `[f32; 3]` (RGB, values in 0.0-1.0)
- Each transform's `color` field (0.0-1.0) indexes into the palette
- Generated by `generate_random_palette()` using cosine gradient functions
- Palette "moods" are randomly selected: dark/dramatic, rich/warm,
  pastel/muted, or high-contrast
- When `taste_engine_enabled`, palettes can be generated by the taste
  engine using learned user preferences
- If `palette` is `None`, the shader falls back to a cosine-based
  procedural palette
